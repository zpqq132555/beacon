# AGENTS Topology

- Keep the root `AGENTS.md` limited to the global entrypoint, key commands, high-value notes, and the AGENTS Map.
- When a directory boundary matches a responsibility boundary, prefer that directory's `AGENTS.md` as the local entrypoint.
- When a directory mixes several stable responsibilities, organize local rules as `local AGENTS.md + [responsibility].md`.
- Responsibility documents for a mixed directory may coexist with child-directory `AGENTS.md` files; this is not an either/or choice.
- Create deeper AGENTS nodes only when responsibilities continue to split in a clear and long-lived way.
